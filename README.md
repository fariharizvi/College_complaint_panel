A Django-based Complaint Management System developed for a college environment, enhanced with Psychological Language Analysis to automatically prioritize complaints based on emotional intensity, stress, and urgency.
This system helps authorities identify critical complaints faster, especially those involving mental distress, safety, or harassment.

Features:

1. User Features
1.1 User Registration & Login
1.2 Submit complaints with title, category, and description
1.3 View complaint status and admin replies
1.4 Automatic complaint priority assignment
1.5 Profile management

2. Psychological Language Analysis
2.1 Detects emotional indicators like stress, fear, anger
2.2 Calculates a stress score
2.3 Assigns priority automatically:
 -Low
 -Medium
 -High
 -Critical

3. Admin Features
3.1 Priority-based complaint dashboard
3.2 Color-coded priority display
3.3 Emotion & stress score visibility
3.4 Complaint status update (Pending / In Progress / Resolved)
3.5 Admin reply system
3.6 Automatic email notification on complaint updates
